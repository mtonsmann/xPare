# Changelog

All notable user-facing changes to xPare are documented here. The format
follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/); versions
follow semver over the public surface — the C ABI, the config schema, and the
CLI flags (see [`docs/release-model.md`](docs/release-model.md), "Versioning
and stability").

## 1.0.0 — 2026-06-10

First stable release. Content-identical to 1.0.0-rc.3, which passed the full
release gauntlet: the SHA-exact fuzz gate, a clean cross-machine launch test,
and the first end-to-end Developer ID sign -> notarize (Accepted) -> staple ->
Gatekeeper-accepted run. From this release the public surface is frozen under
semver: the C ABI (v3), the config schema (v2), and the CLI flags. See the
release-candidate entries below for everything that landed on the way here.

## 1.0.0-rc.3 — 2026-06-10 (release candidate)

### Fixed

- **The rc.2 preview app only ran on the machine that built it**: `core-ffi`
  emitted both a static archive and a dylib, and the linker silently preferred
  the dylib — whose install path pointed into the build tree — over the static
  archive the docs promised. The dylib crate-type is gone, the Swift package
  links `libxpare_ffi.a` as an explicit linker input (immune to stale dylibs),
  and packaging now fails closed if the binary has any non-OS dynamic library
  dependency. Verified by launching the packaged app with the build tree
  removed.

## 1.0.0-rc.2 — 2026-06-10 (release candidate)

### Fixed

- **Release packaging**: the unsigned-preview/dist path built a SwiftPM product
  name derived from the bundle name (`xPareApp`), which no longer exists after
  the project rename — the product is `XPareApp`. The v1.0.0-rc.1 tag's release
  run failed at this step, so rc.1 produced no artifacts; the packaging path is
  now exercised end-to-end before tagging.

## 1.0.0-rc.1 — 2026-06-10 (release candidate)

First public release candidate, under the project's new name.

### Changed

- **Project renamed SafetyStrip → xPare.** The C ABI was renamed with it in one
  coordinated pre-1.0 event (`ss_*`/`SsStatus`/`SS_STATUS_*` →
  `xp_*`/`XpStatus`/`XP_STATUS_*`), bumping `XP_ABI_VERSION` 2 → 3 while the
  ABI has zero external consumers. The ABI is frozen from 1.0 onward.
- **Default global hotkey is now ⌃⌥⌘V** (was ⌥⌘V), and the shortcut is
  recordable in Settings. Combos persisted by older builds are preserved
  verbatim. A hotkey that fails to register is surfaced in the menu and in
  Settings instead of failing silently.
- Clipboard-too-large refusals now distinguish rich content over the size
  limit from oversized plain text in the menu status line.

### Added

- `XP_STATUS_ERR_UNSUPPORTED_CONFIG_VERSION` (ABI status 5): a config
  schema-version mismatch is now distinct from malformed config JSON, so the
  app can say "settings format is newer than this core supports — update the
  app".
- **Continuous mode skips concealed clipboard content**: pasteboards declaring
  `org.nspasteboard.ConcealedType` / `TransientType` / `AutoGeneratedType`
  (password managers and the like) are left untouched without being read.
  Manual triggers still process them.
- **Launch at login** toggle in Settings (macOS `SMAppService`).
- A version row in the menu, and inline validation for line-operation values
  in Settings.
- Release integrity assets: notarized + stapled arm64 builds, per-asset
  `.sha256` files, an aggregate `SHA256SUMS`, build-provenance attestation,
  and an SPDX SBOM. Releases are published as drafts (prereleases marked).
- License texts: `LICENSE-MIT` and `LICENSE-APACHE` now ship verbatim at the
  repository root (the README and crate metadata had claimed dual licensing
  without the texts).
- A declared, CI-checked minimum supported Rust version (1.85) for building
  from source; MSRV bumps are treated as minor releases.

### Fixed

- **HTML→Markdown nested-list output amplification**: deeply nested lists
  could emit quadratically large output (18 KB of adversarial input produced
  8 MB), bypassing the per-operation growth envelope that config validation
  relies on. Rendered list indentation is now clamped (structure tracking and
  ordered numbering stay exact at any depth), and the operation's declared
  worst-case growth factor was re-derived honestly (3 → 5).
- A clipboard write that fails no longer destroys the clipboard silently: the
  failure is surfaced in the menu and is not recorded as a successful write.

### Security

- Hardened the in-memory hygiene posture: op output accumulators that can
  outgrow their initial capacity now wipe superseded allocations before the
  allocator reclaims them, and the remaining best-effort gaps are inventoried
  honestly in [`SECURITY.md`](SECURITY.md).
- The release fuzz gate and diff-scoped mutation testing run again: an
  environment-variable rename left over from the project rename made both
  silently vacuous in CI.

[Earlier development happened pre-rename and pre-versioning; this is the first
